Types

Depending on the action required, different button types are provided. Combined with button styles, these types help create visual action hierarchy.

Primary

This button style is used to indicate the primary action on a page. In practice you should declare no more than one button as the primary button per a page. The primary style is the loudest button styles we have. To add the primary style, add the ._primary modifying class to .button.

{% highlight html linenos %} … {% endhighlight %}

Secondary

This button style is used to indicate important, but secondary actions on a page. To add the secondary style, add the ._secondary modifying class to .button.

{% highlight html linenos %} … {% endhighlight %}

Confirm

This button style is used to indicate a major action, typically within dedicated workflows such as completing a purchase (checkout), publishing an item, or updating an existing item. To add the confirm style, add the ._confirm modifying class to .button.

{% highlight html linenos %} … {% endhighlight %}

Warning

This button style is warn users about a potential negative action such as removing a post, removing a team member, or deleting their account. To add the warning style, add the ._warning modifying class to .button.

Styles

By default all button types have a non-white background and border color. Depending on the community, the button may or may not have an inset box-shadow. However there are times when a particular button type is desired, but we may want the style to be slightly altered based on the page, section or UI context it sits within.

Clear

This is our base button style. It mimics the a "link" style, but providing a larger click area than a normal textual link. This the most understated style available and should be used on low-level, non-important actions. The goal here is to downplay the button’s visual importance.

{% highlight html linenos %} {% endhighlight %}

Outline

The outlined button style is most like the secondary button type in visual styling, but visually downplayed slightly. Here we maintain a border color, but we abandon the background and inset shadow colors. This stylistic variation is to be used when a simpler button style is desired—especially in concert with the primary button type. This style works best with actions that are secondary or tertiary in importance, or when a button type used at a larger size draws undesired attention within a layout.

Ordering

To maintain product consistency, buttons should maintain the following layout ordering:

Within a row

Most button groups should be ordered from the most important to the least important action, left to right.

{% highlight html linenos %} {% endhighlight %}

Within a column

Sometimes the layout dictates that buttons need to be stacked on top of each other. Again, these buttons should be stacked from the most important to the least important, top to bottom.

{% highlight html linenos %}
{% endhighlight %}

Within a header

Sometimes the best place for a series of actions is in the same area as the title. In these cases, the buttons should be pulled to the right. Within this instance, the button order should be reversed with the most important action to the far right and the least important action to the far left.
