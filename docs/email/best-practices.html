Writing HTML & CSS

Tables(!) and inline CSS(!!) and Microsoft Word as a rendering engine(!!!)? The intricacies of email coding are a little different from the web. Here are a few rules of thumb to help avoid some of HTML email's common gotcha's.

  • Inline CSS instead of classes (write CSS as you go)
  • class used only for progressive enhancements like @media and :hover states.
  • table instead of div for layout.
  • When using tables don’t forget border="0" cellpadding="0" cellspacing="0" role="presentation".
  • px< instead of em, rem, or %.
  • Except % for container and img widths.
  • align instead of float, grid, or flexbox.
  • padding instead of margin for spacing in <td>s.
  • But why? Margins aren’t fully supported by Hotmail & Outlook.com.
  • PNG, GIF, or JPG (@2x) instead of SVG or Base64 encoded images.
  • #ffffff instead of #fff.
  • Hybrid instead of responsive for mobile.
  • Recommend against using a CSS inliner (don’t worry if you don’t know what that is).
  • Reference Campaign Monitor for support on a specific HTML or CSS property.
