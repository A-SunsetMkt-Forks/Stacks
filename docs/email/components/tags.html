For when the color of tags can be customizes or user defined.

tag-name tag another-tag-name

Themed color are defined using a dark theme color (used for text color) and applying that same color at 15% opacity as the background. However not all email clients support background opacity, so this explains our “clever hack” around those limitations.

Initially we define a neutral gray color #EFF0F1 . That's mainly for Windows Outlook and Yahoo (web and mobile), who will display themed tags with a light gray background and the text set in the theme color.

But for email clients that support opacity, we can use the theme color at 15% as a longer hex code like #2b8a3e15. But when we apply a background opacity to both the border and background, we get a visible border. But for tagfs, the border is only used to give this button some padding (because some email clients don’t apply padding to text elements) and the background color is already what we need. Applying background-color: transparent; would be ideal, but Outlook reads that as nothing and doesn’t apply the background width. So we're making the team color appear at 1% for the border. Technically it’s there (so Outlook doesn’t collapse our tag’s padding) but the 1% opacity is imperceivable to humans.
