Introduction

If you only need a single plain CSS file that can be dropped into your fresh project to start using Stacks, you probably won’t care about this section. stacks.css (also available in a minified version) may be all you need.

If you compile the LESS files inside your own project and/or need to customize certain things, this document is for you.

Static part and dynamic part

Stacks consists of a static and a dynamic part. In order to understand the difference, it helps to explain why this separation exists.

Stacks is used on all Q&A sites in the Stack Exchange network, including Stack Overflow. Many of those sites have their own themes, e.g. with different link colors and different button styles. Large parts of Stacks are agnostic to these differences – for example, the atomic class .ps-fixed does the same thing regardless of what site it’s used on. But some parts of Stacks need to behave differently depending on what site they’re used on – for example, the background color of the .s-btn component differs per-site.

The parts of Stacks that are always the same across all Stack Exchange sites are what we call the static part. The parts of Stacks that are configurable based on the Q&A site’s theme are called the dynamic part.

The static parts of Stacks are included in all network pages as a single CSS file that’s identical everywhere. The dynamic parts, on the other hand, are compiled into the site’s primary.css file.

Configuration

While Stacks is pretty opinionated, it does have some configuration options. Most of these options configure the different appearances of the various themes in the dynamic part of Stacks (see above), for example the button colors. There are a few (currently two) options that also configure the static parts; these are settings that may be different per project (e.g. Talent uses a different CSS reset than Q&A), but are identical across all Q&A network sites (all Q&A sites use the same CSS reset).

Compiling a customized version of Stacks happens by including stacks.less (or, depending on your need, stacks-dynamic.less and/or stacks-static.less) in your LESS file and overriding the default configuration options. These options are contained in two different mixins.

For example, to compile the dynamic parts of Stacks into your own CSS file, but with Roboto as the font family and pink hyperlinks, you would do something like this:

{% comment %}the rouge highlighter doesn't support LESS :( {% endcomment %} {% highlight css linenos %} @include "path/to/stacks-dynamic.less"; #stacks-config { @body-type-face: 'Roboto', sans-serif; @link-color-regular: magenta; @link-color-hover: lighten(magenta, 20%); } {% endhighlight %}

To compile the complete LESS with its default values, except that you don’t want a CSS reset compiled in, you would do this:

{% highlight css linenos %} @include "path/to/stacks.less"; #stacks-config-static { @css-reset: none; } {% endhighlight %}
