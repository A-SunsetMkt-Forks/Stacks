{% header "h2", "Examples" %}

Link unfurls, also referred to as oneboxes, are those handy previews that automatically appear when you paste a link into a post. This approach has been popularized by Slack, Twitter, Facebook, etc. Unfurled previews are handy because they give everyone a glimpse of what’s been linked to. Link unfurling should be tailored to fit the content received from each third party. It might be appropriate to use atomic utility classes for further customizing unfurl previews, but Stacks provides a few reusable patterns:

make the .XXX-unset classes work in IE
Pull request merged by hellohynes on on Jun 30, 2018 in StackExchange / Stacks

Stacks is providing a bunch of .FOOBAR-unset atomic classed. All of these classes literally set the unset value on the corresponding properties.

Since unset doesn't exist in IE, which Stacks has to support, this doesn't work. Fortunately, unset is just minor syntactic sugar for either inherit or initial, depending on whether the property is inherited or not.

Therefore, we can continue providing the Stacks user with ...-unset classes that do the right thing, and that also work in IE.

It does not change the class names; they still contain the word "unset", for the following reasons:

  • The semantics are identical; these class do have the same effect as unset, except that they work in IE.
